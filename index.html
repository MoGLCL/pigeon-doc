01 · Prepare

Requirements

Install the base tools once. The exact runtime topology changes with the target, but the application requirements stay predictable.

JSNode.js 22 LTSIncludes npm for dependency and script management.
DBPostgreSQL 16+Local, containerized or a secure managed instance.
GitGitRequired for source updates and the separate OpenWA checkout.
TLSHTTPS domainRequired in production for OAuth callbacks and secure sessions.
OpenWA is a separate service

Pigeon connects to an existing rmyndharis/OpenWA installation. Regular users only scan QR codes; infrastructure configuration stays with the owner.

03 · Install

Windows local

Use the Pigeon-Windows-Local package for a native setup without Docker.

Create the local environment file

Copy the safe template, then replace every placeholder before migration.

PowerShell
Copy-Item .env.example .env

Install and prepare PostgreSQL

Set DATABASE_URL, then generate the client and apply the development migrations.

PowerShell
npm.cmd install
npm.cmd run db:generate
npm.cmd run db:migrate
npm.cmd run db:seed

Start the complete local stack

The bootstrap checks the configured OpenWA service before starting Next.js and Socket.IO.

PowerShell
npm.cmd run dev

04 · Install

Linux + Docker

Use the Pigeon-Linux-Docker package for a reproducible Linux production deployment.

Prepare production values

Set the public HTTPS origin, fresh secrets and a strong PostgreSQL password.

Shell
cp .env.example .env

Build and start the stack

Compose persists PostgreSQL and application media in named volumes.

Shell
docker compose build
docker compose up -d
docker compose ps

Seed the first owner once

Production migrations run automatically before startup. Seed only on the first deployment.

Shell
docker compose exec app npm run db:seed
Do not expose PostgreSQL publicly

Remove the database port mapping when remote administration is not required. Use the internal Compose network from the app.

05 · Install

Private hosting

Use the dedicated private-hosting source on a Node-capable VPS or control panel. It includes a PM2 ecosystem file and production environment template.

Upload source only

Never upload .env, node_modules, .next, database dumps or user media from another machine.

Install, migrate and build

Shell
npm ci
npm run db:generate
npm run db:deploy
npm run db:seed # first installation only
npm run build

Start behind a reverse proxy

Forward HTTP and WebSocket traffic to port 3000, then use the final HTTPS origin for APP_URL and AUTH_URL.

PM2
pm2 start ecosystem.config.cjs

06 · Configure

Environment & secrets

Environment variables bootstrap the first start. After that, owner-managed integration values are stored in PostgreSQL and secret values are encrypted.

VariablePurposeRequired
DATABASE_URLPostgreSQL connection string.Always
AUTH_SECRETSigns and protects authentication state.Always
ENCRYPTION_KEYEncrypts integration secrets and message content.Always
APP_URL / AUTH_URLFinal public application origin.Production
CRON_SECRETAuthenticates scheduled-job requests.Production
OWNER_*Creates the initial owner during the first seed.First install
The encryption key is not replaceable casually

Back it up securely. Losing it makes previously encrypted tokens, settings and message content unrecoverable.

07 · Connect

Facebook & Messenger

Configure one Meta application at platform level. Each user then connects Facebook and selects only the Pages they want in their workspace.

Create the Meta application

Add the Facebook Login and Messenger/Page products required by your approved use case.

Set the callback

Automatic mode derives it from the public APP_URL. Manual mode uses the exact callback saved by the owner.

Reconnect after permission changes

Existing tokens must be reauthorized to receive added permissions such as full Page content and insights.

Expect privacy thresholds

Meta may withhold audience demographics or retire insight metrics. Pigeon displays an unavailable state instead of generating fake values.

08 · Connect

WhatsApp / OpenWA

OpenWA is operated once by the platform owner. Users only create an account name and scan its QR code.

Managed local service

On Windows, Pigeon can start an existing OpenWA checkout when automatic startup is enabled.

OPENWA_AUTO_START=true

External service

For Docker and hosting, run OpenWA separately and configure a private network or HTTPS base URL.

OPENWA_AUTO_START=false

Operator-only secrets

The API key and infrastructure fields never appear in regular user settings or browser responses.

Encrypted in DB

11 · Verify

Verification checklist

A build is not a release until the code, database and live integration paths all pass.

Release checks
npm ci
npm run db:generate
npm run typecheck
npm test
npm run build
npm audit --omit=dev
  • HTTPS callback matches the owner configuration.
  • WebSocket messages reach the active browser.
  • Password-reset email is delivered correctly.
  • One WhatsApp QR session connects successfully.
  • Facebook Page reconnect imports real posts.
  • Admin roles see only authorized sections.
12 · Protect

Backups & recovery

Back up the database and persistent media as one recovery point. A database without its matching encryption key or uploads is incomplete.

01PostgreSQLAutomated encrypted dumps with tested retention.
02Uploads & mediaPersist and copy the application data volume.
03Encryption keyStore separately in a secrets manager.
04Restore drillTest a complete restore before an incident.
13 · Diagnose

Troubleshooting

Start with the smallest observable layer: process, database, integration service, then browser delivery.

The app does not start

Confirm Node.js 22, the database connection, generated Prisma client and applied migrations. Run npm run typecheck before rebuilding.

WhatsApp stays disconnected

Check the owner-configured OpenWA health, base URL and API key. On hosting, ensure OpenWA runs on infrastructure that supports a persistent browser process.

Facebook data is incomplete

Reconnect the Page after permission changes, then check the live warning returned by Meta. Insight and demographic availability is controlled by Meta.

Realtime updates are delayed

Confirm reverse-proxy WebSocket upgrades, webhook delivery, the Socket.IO connection and scheduled sync process.
